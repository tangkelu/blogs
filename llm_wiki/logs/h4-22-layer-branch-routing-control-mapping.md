## Four Controlled Claim Families

The first stable `22-layer` routing surface should be treated as four controlled claim families only:

1. `standards_family_or_hierarchy_identity`
2. `qualification_or_listing_identity`
3. `acceptance_workflow_identity`
4. `supplier_conformance_assertion`

Interpretation rule:

- these families are adjacent in standards-heavy drafting
- adjacency does not merge them
- none of them becomes reusable threshold, approval, release, or supplier-proof authority by default

## Default Branch Order

When a future `22-layer` sentence or draft fragment is reviewed, the controller should ask these questions in order:

1. Is this hierarchy identity, not threshold identity?
2. Is this listing or qualification ecosystem identity, not current status or release authority?
3. Is this workflow naming, not accepted-result or threshold logic?
4. Is this trying to prove supplier conformance from public-facing context?

If the answer to any later question is yes, stop at the earlier safe route.

Do not let the draft continue upward into proof-like language.
