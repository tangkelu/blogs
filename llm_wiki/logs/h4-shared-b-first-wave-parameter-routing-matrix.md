# H4 Shared B First-Wave Parameter Routing Matrix

Date: 2026-04-26

## Purpose

This file converts `h4-shared-b-capability-parameters-first-wave-queue.md` into an execution-grade routing matrix for the first shared `B` parameter wave.

It covers only these seven parameter classes:

1. `trace/space`
2. `minimum mechanical drill`
3. `minimum laser via`
4. `aspect ratio`
5. `annular ring`
6. `impedance tolerance`
7. `registration tolerance`

This file is:

- a routing/control matrix
- an execution note for shared blocked parameter classes
- a branch-pressure map for future evidence handling

This file is not:

- a numeric recovery result
- a recovered capability table
- permission to reconstruct geometry, tolerance, or acceptance numerics
- proof that public reusable numeric readiness has improved

Public reusable numeric readiness remains unchanged after this file.

## First Named Blocked Branches

The first named blocked branches for this matrix are:

- `20-layer`
- `22-layer`

They stay first because they still carry the strongest combined pressure for:

- geometry-capability leakage
- standards-threshold leakage
- workflow-to-proof collapse
- supplier-capability implication

`14-layer`, `18-layer`, and `24-layer` may later inherit routing outcomes if acceptable authority ever appears.

That possible reuse is downstream only.

This matrix does not itself recover numbers.

## Matrix Use Rule

For every parameter below, the controller should decide only:

- where demand currently comes from
- what blocker shape currently prevents reusable numeric writing
- what authority would be acceptable in principle
- what refusal posture applies if that authority is missing

This is routing, not numeric recovery.

## Global Non-Reuse Boundary

The following source shapes do not qualify by default as reusable numeric authority for this matrix:

- supplier-scoped proof pages or factory marketing summaries
- standards-threshold pages, TOCs, clause-family summaries, or hierarchy pages
- derivative comparison tables
- broad process-overview pages without explicit parameter authority
- draft demand from `20-layer`, `22-layer`, or adjacent blogs

Those source shapes may help boundary control.

They do not become reusable numeric basis for shared `B` parameters.

## Parameter Routing Matrix

### 1. `trace/space`

Main demand branches:

- `20-layer`
- `22-layer`
- `14-layer`
- `18-layer`

Current blocker shape:

- often arrives as generic high-density capability shorthand rather than named-source geometry authority
- often drifts from supplier-scoped capability claims into reusable numeric table pressure
- often gets strengthened by standards-threshold adjacency even when no public threshold actually authorizes the geometry

Acceptable authority requirement:

- a primary source must expose the parameter directly as a reusable numeric field with clear scope and condition
- the source must be strong enough to show whether the number is product-, process-, lot-, or supplier-scoped
- if the source is supplier-scoped, it must stay supplier-scoped and dated rather than being generalized into a shared reusable class

Default refusal posture if authority missing:

- refuse numeric recovery
- keep only non-numeric density/manufacturability context
- route supplier-specific capability wording to `hold_for_supplier_evidence`

### 2. `minimum mechanical drill`

Main demand branches:

- `20-layer`
- `22-layer`
- `18-layer`
- `24-layer`

Current blocker shape:

- often collapses stack difficulty, hole-quality pressure, and factory capability into one unsupported minimum
- often appears through routing-rule summaries that do not show acceptable authority for transferable reuse
- can be misread from standards or acceptance context even when those sources do not publish reusable drill minima

Acceptable authority requirement:

- a primary source must publish the minimum field directly with clear test or manufacturing scope
- the scope must distinguish general material/process identity from actual drill-capability identity
- standards-family or clause-family visibility alone is not acceptable authority for the number

Default refusal posture if authority missing:

- refuse the minimum drill number
- retain only non-numeric through-hole difficulty or manufacturability wording
- do not let standards pages or supplier summaries backdoor a drill-capability table

### 3. `minimum laser via`

Main demand branches:

- `20-layer`
- `22-layer`
- `18-layer`

Current blocker shape:

- the strongest pressure comes from `20-layer` HDI and build-up language
- public any-layer or method vocabulary gets repeatedly overextended into unsupported geometry proof
- supplier-scoped microvia claims are especially likely to be mistaken for reusable public authority

Acceptable authority requirement:

- a primary source must expose the laser-via minimum directly and with explicit scope
- the authority must separate method identity, material identity, and actual capability identity
- dated supplier records may later support a narrow supplier-scoped statement, but they still would not become generic shared numeric authority by default

Default refusal posture if authority missing:

- refuse numeric recovery
- keep only architecture, build-up, or method-name context
- route any supplier microvia-capability statement to `hold_for_supplier_evidence`

### 4. `aspect ratio`

Main demand branches:

- `20-layer`
- `22-layer`
- `18-layer`
- `24-layer`

Current blocker shape:

- often used as a compressed shop-capability proxy for drill, plating, thickness, and yield
- often arrives as a transferable rule without acceptable public authority for the exact number
- easily absorbs hidden threshold logic from standards-heavy or acceptance-heavy drafts

Acceptable authority requirement:

- a primary source must state the aspect-ratio field directly and in a way that makes reuse conditions explicit
- the authority must be narrow enough to show whether the number belongs to a named process, product family, supplier, or dated build context
- standards-threshold or acceptance-style pages are not acceptable numeric authority for this class

Default refusal posture if authority missing:

- refuse aspect-ratio numerics
- keep only non-numeric depth/difficulty/manufacturability context
- do not convert process-sensitivity wording into a reusable capability ratio

### 5. `annular ring`

Main demand branches:

- `22-layer`
- `20-layer`
- `14-layer`
- `24-layer`

Current blocker shape:

- this class is the cleanest bridge from capability pressure into standards-threshold leakage
- clause-family visibility around annular ring repeatedly tempts reconstruction of hidden criteria
- supplier-scoped quality claims can also be misread as generic annular-ring authority

Acceptable authority requirement:

- a primary source must expose reusable annular-ring values directly, not just show that annular ring is a standards topic
- the authority must distinguish clause-family existence from the actual numeric criterion
- supplier-scoped records, if ever used, must remain dated and supplier-scoped rather than being promoted into shared reusable numerics

Default refusal posture if authority missing:

- refuse annular-ring numerics
- keep only non-numeric registration/manufacturability pressure context
- explicitly reject standards-threshold pages, addendum summaries, and supplier proof pages as reusable number basis

### 6. `impedance tolerance`

Main demand branches:

- `20-layer`
- `22-layer`
- `14-layer`
- `18-layer`
- `24-layer`

Current blocker shape:

- commonly collapses fabrication control, verification workflow, and SI outcome into one number
- often appears through marketing or shop-capability wording instead of acceptable authority
- is especially vulnerable to false promotion from standards/process language into reusable tolerance claims

Acceptable authority requirement:

- a primary source must expose the tolerance field directly with clear scope and condition
- the source must distinguish design target, fabrication capability, and verification/reporting posture
- supplier-scoped tolerance proof cannot be reused as shared generic authority unless a later workflow explicitly authorizes narrow supplier-only handling

Default refusal posture if authority missing:

- refuse impedance-tolerance numerics
- keep only non-numeric impedance-control or verification-process context
- route supplier-specific tolerance promises to `hold_for_supplier_evidence`

### 7. `registration tolerance`

Main demand branches:

- `22-layer`
- `20-layer`
- `18-layer`
- `24-layer`

Current blocker shape:

- frequently implied indirectly through high-layer density, lamination sensitivity, or alignment difficulty wording
- often appears without any source that actually publishes reusable registration tolerance values
- can be falsely strengthened by standards or inspection workflow pages that discuss dimensional control without exposing the numeric threshold

Acceptable authority requirement:

- a primary source must expose registration tolerance directly as a reusable parameter with explicit scope
- the source must be stronger than workflow or inspection vocabulary alone
- standards-family visibility, workflow naming, or supplier process-discipline marketing are not acceptable numeric authority

Default refusal posture if authority missing:

- refuse registration-tolerance numerics
- keep only non-numeric alignment, lamination, or dimensional-control context
- do not treat registration-sensitive process wording as recovered tolerance authority

## Cross-Parameter Execution Rules

Apply these rules across all seven classes:

1. `20-layer` and `22-layer` remain the first named blocked branches unless a later evidence pass proves otherwise.
2. Do not describe this matrix as if it recovered any numeric class.
3. Do not let supplier-scoped proof pages become shared reusable numeric authority.
4. Do not let standards-threshold pages, TOCs, clause lists, or addendum summaries become reusable numeric authority.
5. If a later narrow exception appears, document it as a class-specific exception rather than as a wave-wide unlock.
