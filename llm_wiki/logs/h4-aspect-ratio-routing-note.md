Date: 2026-04-26

## Purpose

This file is the `H4` Lane 1 Phase B single-parameter routing note for `aspect ratio`.

It exists to standardize how `aspect ratio` claims should be routed before any attempt is made to recover a shared reusable numeric field.

This file is:

- a routing/control note
- a single-parameter blocker map
- a demand-branch execution note for Lane 1 Phase B

This file is not:

- a numeric recovery result
- a recovered capability table
- permission to restore aspect-ratio numbers from weak or indirect sources
- proof that public reusable numeric readiness improved

Public reusable numeric readiness remains unchanged after this note.

## Current Blocker Shape

The current blocker shape for `aspect ratio` is stable and unchanged:

- the corpus has difficulty and process-sensitivity framing, but not acceptable shared reusable numeric authority
- `aspect ratio` is often used as a compressed proxy for drill, plating, thickness, yield, and shop capability rather than as a directly authorized reusable parameter
- process-sensitivity prose can explain why deep drilling or heavy multilayer structures are harder, but it does not authorize a transferable shared numeric ratio
- standards-threshold and acceptance-heavy materials can mention related criteria context, but they do not publish a safe shared reusable `aspect ratio` number for this repo
- supplier-scoped proof can mention specific ratios, but those claims remain supplier-scoped and non-transferable by default
- derivative summaries and routing heuristics often present `aspect ratio` as if it were a universal shop rule without acceptable source authority

The key split that must remain intact:

- `aspect ratio` is a fabrication capability parameter, not a standards-threshold surrogate
- it is separate from acceptance or qualification handling
- it is separate from supplier-scoped dated proof
- it is separate from process-sensitivity prose

## Acceptable Authority Requirement

`Aspect ratio` should be treated as shared reusable only if acceptable authority actually exists.

Minimum requirement:

- a primary source must expose the aspect-ratio field directly
- the scope must be explicit enough to show whether the ratio is shared reusable, supplier-scoped, process-scoped, product-scoped, or build-context-specific
- the source must be narrow enough to distinguish named capability identity from general manufacturability commentary
- the source must make reuse conditions explicit rather than forcing reconstruction from heuristics, prose, or threshold-adjacent material

Not acceptable as shared reusable numeric authority:

- standards-threshold pages
- acceptance pages
- supplier-scoped proof pages
- dated supplier capability records used outside their supplier scope
- process-sensitivity prose
- clause-family or hierarchy summaries
- derivative comparison tables
- generic overview pages without direct parameter authority

If a future source is supplier-scoped and dated, it may support a narrow supplier-only handling lane.

It still does not become shared reusable numeric authority by default.

## Default Refusal Posture

If acceptable authority is missing, default posture is:

- refuse the exact `aspect ratio` number
- keep only non-numeric depth/difficulty/manufacturability context
- reject standards-threshold, acceptance, supplier-proof, and process-sensitivity materials as numeric basis
- preserve separation between capability routing and standards/acceptance routing
- route any supplier-specific ratio claim away from shared reusable handling

Default refusal wording should bias toward:

- `aspect-ratio capability depends on source-specific scope`
- `public reusable aspect-ratio authority is not yet established`
- `non-numeric manufacturability context may be kept, but the numeric ratio remains blocked`

## Non-Reuse Boundary

This note fixes the non-reuse boundary for `aspect ratio`.

The following do not qualify as shared reusable numeric authority:

- standards-threshold pages
- acceptance pages

- process-sensitivity prose
- standards hierarchy pages

These sources may still help with:

- vocabulary control
- claim splitting

- refusal routing

They do not authorize a shared reusable aspect-ratio number.

## Routing Use Rule

When a draft asks for `aspect ratio`, the controller should decide only:

- which branch is creating demand
- whether the claim is really a capability-ratio claim rather than a standards, acceptance, or supplier-proof claim
- whether acceptable direct authority exists
- whether the claim must remain non-numeric or be routed to a narrower supplier-scoped lane

This note is routing only.

It is not numeric recovery.
