## Queue Output Contract

Every `H3` queue file should explicitly include:

1. queue purpose
2. current baseline
3. inventory by draft area or risk cluster
4. source-policy split:
   - `metadata_only`
   - `public_threshold_available`
   - `controlled_exclusion`
5. default disposition by cluster
6. allowed residual wording classes
7. blocked numeric or assertion classes
8. queue completion criteria

If `public_threshold_available` is empty, the file must say so directly.

- whether readiness changed
- what still remains blocked

## Stop Conditions

Stop and record `readiness unchanged` if the latest pass only does one or more of the following:

- splits a blocker into narrower buckets
- improves blacklist precision
- improves residual wording rules
- improves prompt-safe retrieval without stronger authority

Do not record any of the following unless separately proven:

- threshold recovery
- supplier approval or compliance proof
- qualification-package proof
- accepted-lot or release authority
- high-density numeric readiness
