# Source Summary

## What It Covers

- USB Type-C receptacle, plug, cable, and detection-mechanism scope
- Relationship between USB Type-C, USB 2.0, USB 3.2, USB4, and USB Power Delivery specifications
- Normative versus informative specification framing

## Why It Matters

- Gives `type-c-charger` a primary USB-IF source for USB Type-C as a connector and cable ecosystem rather than relying only on internal charger pages.

## Extraction Notes

- Safe for source identity, scope, and vocabulary such as `USB Type-C receptacle`, `plug`, `cable assembly`, `detection`, and `USB Power Delivery optimized for the connector`.
- Do not extract exact pinout, current, cable, termination, mechanical, or test values without a separate parameter extraction pass.
- Do not treat this 2019 release as the latest current revision without checking the USB-IF document library and compliance updates.
