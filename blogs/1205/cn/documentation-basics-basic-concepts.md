## 1. 执行摘要：从混乱到可控，构建硬件研发的“单一事实来源”

对于初创研发团队而言，速度与创新是生命线。然而，在追求快速迭代的过程中，PCB 设计文档（PCB Documentation）的缺位或混乱，往往成为项目延期、预算超支和质量问题的隐形杀手。您是否经历过以下痛点？

*   **版本失控**：工程师 A 的 BOM 与工程师 B 的装配图不匹配，导致采购了错误的元器件。
*   **沟通黑洞**：设计意图仅存在于工程师的脑海中，口头传递给工厂，导致生产出来的电路板与预期功能大相径庭。
*   **返工陷阱**：因缺少明确的 DFM（Design for Manufacturability）检查，设计文件在投产后被工厂驳回，反复修改，浪费了宝贵的研发时间。
*   **知识孤岛**：核心设计参数和决策依据没有文档化，一旦人员变动，项目几乎需要从零开始。

这些问题并非个例，而是缺乏标准化 **pcb documentation basics** 体系的必然结果。本白皮书由 **HilPCBPCB Factory (HILPCB)** 设计赋能团队撰写，旨在为初创团队提供一套可复制、可扩展的 PCB 设计文档与流程框架。

**我们的目标是**：将 PCB 设计文档从一项“事后补救”的繁琐任务，转变为驱动项目成功的核心资产。通过建立统一的术语、标准化的流程、清晰的交付物清单和可量化的 KPI，您的团队将能够：
*   **建立单一事实来源（Single Source of Truth）**：确保所有团队成员（设计、采购、生产、测试）基于同一套准确信息协同工作。
*   **提升一次成功率**：通过系统化的 DFM/DFT 检查，在设计阶段消除制造与测试隐患。
*   **加速产品迭代**：标准化的文档使设计复用、知识传承和新成员上手变得轻而易举。
*   **降低综合成本**：减少因沟通不畅和设计缺陷导致的物料浪费与生产返工。

本白皮书将引导您从基础术语开始，逐步构建覆盖“需求-设计-交付”全流程的文档化体系，最终实现高效、可靠、可扩展的硬件研发流程。

## 2. 术语与结构图谱：统一沟通语言

在建立流程之前，我们必须确保团队内部及与外部供应商之间使用同一种“语言”。不一致的术语是导致误解和错误的主要原因。下表梳理了 PCB 设计文档体系中的核心名词。

| 名词 (Term) | 定义 (Definition) | 对项目的影响 (Impact) | 参考/最佳实践 (Reference/Best Practice) |
| :--- | :--- | :--- | :--- |
| **Schematic (原理图)** | 使用标准化符号表示电路连接关系和元器件的逻辑图。 | 设计的逻辑基础，定义了电路的功能。错误将导致整个项目失败。 | 保持清晰的层次结构，标注关键信号和电源网络。 |
| **Netlist (网表)** | 从原理图导出的文本文件，描述了元器件引脚之间的连接关系。 | 连接原理图与 PCB 布局的桥梁。**Netlist and footprints** 是 PCB 设计的两大基石。 | 导入 PCB 编辑器后，务必进行交叉验证，确保所有连接正确无误。 |
| **Footprint (封装)** | 元器件在 PCB 上的物理尺寸、焊盘布局和丝印标识。 | 封装错误会导致元器件无法安装或焊接不良，是常见的装配失败原因。 | 严格遵循元器件数据手册（Datasheet）或使用 IPC-7351B 标准库。 |
| **Stackup (层叠结构)** | PCB 各导电层与绝缘层的排列顺序、材料、厚度等参数的定义。 | 决定了电路板的阻抗、信号完整性、EMC 性能和制造成本。 | 在设计初期与制造商（如 HILPCB）确认，特别是对于高速或多层板设计。 |
| **BOM (物料清单)** | Bill of Materials。列出组装 PCB 所需的所有元器件的详细清单。 | 采购、仓储和生产的核心依据。错误或不完整的 BOM 会导致生产停滞。 | 包含制造商型号、封装、数量、位号等关键信息。参考我们的 **BOM preparation tips**。 |
| **Gerber Files** | 描述 PCB 各层图像（铜层、阻焊、丝印等）的标准文件格式。 | 提交给 PCB 制造商用于生产裸板（Bare Board）的“蓝图”。 | 使用 RS-274X 或 X2 格式，并用 Gerber 查看器 检查。 |
| **Drill Files (钻孔文件)** | 指定 PCB 上所有钻孔（过孔、安装孔）位置和尺寸的文件。 | 钻孔是实现层间连接和元器件安装的关键。错误将导致电路开路或装配失败。 | 通常为 Excellon 格式，需与 Gerber 文件一同提交。 |
| **DRC (设计规则检查)** | Design Rule Check。软件根据预设规则自动检查设计是否满足制造工艺要求。 | 自动化地发现潜在的制造问题，如线宽/间距过小、安全间距不足等。 | 规则应基于制造商的工艺能力设定，而非使用软件默认值。 |

## 3. 流程框架：从需求到交付的文档化路径

一个稳健的文档体系并非孤立存在，而是深度嵌入在整个设计流程中。我们将 PCB 开发流程划分为六个关键阶段，并明确每个阶段的核心文档产出与活动。

  <!-- Placeholder for a flowchart image -->

**阶段一：需求与规格定义 (Requirements & Specification)**
*   **核心活动**：定义产品功能、性能指标、成本目标、使用环境等。
*   **文档产出**：
    *   **产品需求文档 (PRD)**：描述产品要“做什么”。
    *   **硬件规格书 (HRS)**：将产品需求翻译为具体的电气参数、接口标准、物理尺寸和安规认证要求。这是所有后续设计的最高纲领。

**阶段二：原理图与层叠设计 (Schematic & Stackup)**
*   **核心活动**：进行电路逻辑设计，选择关键元器件，并与制造商初步确定层叠方案。
*   **文档产出**：
    *   **初步原理图 (Schematic Draft)**：完成核心功能模块的设计。
    *   **初步 BOM (Preliminary BOM)**：用于评估成本和关键物料的供货周期。
    *   **层叠设计方案 (Stackup Proposal)**：与 **HILPCB** 这样的专业制造商沟通，获取基于成本和性能的[标准层叠建议](https://hilpcb.com/en/products/multilayer-pcb)。

**阶段三：布局 (Placement)**
*   **核心活动**：根据信号流、热分布和结构约束，在 PCB 上放置元器件。
*   **文档产出**：
    *   **布局规划图 (Placement Plan)**：标注关键模块区域，如电源区、模拟区、高速数字区。
    *   **3D 模型 (STEP/IGES)**：用于与结构工程师进行干涉检查。

**阶段四：布线 (Routing)**
*   **核心活动**：根据设计规则连接所有网络，优化信号完整性和电源完整性。
*   **文档产出**：
    *   **布线约束报告 (Routing Constraint Report)**：记录关键网络（如差分对、时钟线）的阻抗、长度匹配等规则。
    *   **PCB 设计文件 (e.g., .PcbDoc, .brd)**：包含完整布局布线信息的源文件。

**阶段五：设计评审 (Design Review)**
*   **核心活动**：进行内部（同行）和外部（DFM）评审，确保设计质量。
*   **文档产出**：
    *   **设计评审检查表 (Design Review Checklist)**：覆盖电气、机械、制造等方面的检查项。
    *   **DRC/DFM 报告**：软件自动检查报告和来自制造商的专业分析报告。

**阶段六：生产资料交付 (Manufacturing Data Delivery)**
*   **核心活动**：生成并打包所有用于制造和装配的文件。
*   **文档产出**：
    *   **制造文件包 (Fabrication Package)**：Gerber、钻孔文件、层叠说明。
    *   **装配文件包 (Assembly Package)**：BOM、坐标文件 (Pick and Place)、装配图。

## 4. 材料/层叠标准库：构建可靠的物理基础

选择错误的材料或层叠结构，就像在沙地上建造高楼。为初创团队建立一个标准库，可以极大简化决策过程，并确保设计的可制造性。

#### 基线标准库 (Baseline Standard Library)

**1. 单/双面板 (Single/Double Layer)**
*   **典型应用**：低成本消费电子、电源模块、简单的控制板。
*   **标准材料**：FR-4, Tg 130-140°C。
*   **标准铜厚**：1 oz (35µm)。
*   **标准板厚**：1.6mm。
*   **参考教程**：对于初学者，可以从一个简单的 [double layer pcb tutorial](https://hilpcb.com/en/products/single-double-layer-pcb) 开始，掌握基础的布局布线概念。
*   **文档要点**：在制造说明中明确板材、板厚、铜厚、阻焊颜色和丝印颜色。

**2. 四层板 (Four Layer)**
*   **典型应用**：带有微控制器 (MCU) 的物联网设备、中等复杂度的控制系统。
*   **标准叠层 (经典)**：
    *   L1: Signal
    *   L2: GND (Ground Plane)
    *   L3: Power Plane
    *   L4: Signal
*   **优势**：完整的地平面和电源平面提供了优异的信号完整性和 EMC 性能，布线也更加容易。
*   **文档要点**：必须提供明确的层叠结构图，包括每层的功能、介质厚度和铜厚。**HILPCB** 的在线报价系统可以帮助您快速仿真和定制标准叠层。

#### 扩展应用库

*   **高速/高频设计**：需要使用低损耗材料（如 Rogers, Teflon）并进行精确的阻抗控制。层叠文档必须包含目标阻抗值和对应的线宽。
*   **大功率设计**：可能需要使用[厚铜 PCB](https://hilpcb.com/en/products/heavy-copper-pcb)（≥2 oz），层叠文档需特别注明铜厚要求。
*   **高密度设计 (HDI)**：涉及微盲埋孔，层叠结构更为复杂，需要与制造商进行深度技术沟通。

## 5. 模块化布局/布线规则：将经验转化为标准

将设计经验文档化为具体规则，是确保团队设计质量一致性的关键。以下是针对不同功能模块的规则建议。

*   **时钟与高速信号 (Clock & High-Speed Signals)**
    *   **规则 1 (阻抗控制)**：所有高速信号线（如 USB, Ethernet, MIPI）必须进行阻抗匹配。使用 阻抗计算器 工具，根据层叠参数计算目标线宽。
    *   **规则 2 (路径最短)**：保持走线简短、直接，避免桩线 (stub)。
    *   **规则 3 (差分对)**：差分信号对内等长、等距布线，避免在中间穿越分割区域。

*   **电源分配网络 (Power Delivery Network - PDN)**
    *   **规则 1 (路径低阻)**：电源路径应宽而短。对于大电流路径，可以使用平面或更宽的走线。可以参考一些 **trace width calculator intro** 来理解电流与线宽的关系。
    *   **规则 2 (去耦电容)**：去耦电容应尽可能靠近 IC 的电源引脚放置，连接路径要短。
    *   **规则 3 (星形接地)**：在模拟和数字混合电路中，采用单点接地或星形接地，避免噪声耦合。

*   **模拟电路 (Analog Circuits)**
    *   **规则 1 (隔离)**：将模拟电路区域与数字电路和电源电路物理隔离。
    *   **规则 2 (保护环)**：在敏感的模拟信号线周围布设地线保护环 (Guard Ring)。
    *   **规则 3 (独立地)**：为模拟部分提供一个干净、独立的接地平面或接地路径。

*   **功率器件 (Power Components)**
    *   **规则 1 (散热)**：大功率器件下方应铺设大面积铜皮，并增加散热过孔 (Thermal Vias) 连接到内层散热平面。
    *   **规则 2 (安全间距)**：高压部分必须满足安规要求的电气间隙 (Clearance) 和爬电距离 (Creepage)。

## 6. 文档清单：确保交付万无一失

一个完整的生产资料包应该像一份精美的食谱，让任何制造商都能准确无误地烹饪出您的“设计大餐”。下面是一个基础的 **pcb design checklist beginner** 团队可以遵循的交付清单。

| 文件类别 | 文件清单 | 用途与关键检查点 |
| :--- | :--- | :--- |
| **PCB 裸板制造** | 1. **Gerber Files** (所有铜层, 阻焊层, 丝印层, 边框层) <br> 2. **NC Drill File** (钻孔数据) <br> 3. **Stackup Drawing** (层叠图) <br> 4. **Fabrication Notes** (制造说明) | **用途**: 用于生产 PCB 裸板。 <br> **检查点**: 格式是否为 RS-274X/X2？单位、精度是否正确？层叠图是否与设计文件一致？特殊工艺（如金手指、阻抗控制）是否在说明中注明？ |
| **元器件采购** | 1. **Bill of Materials (BOM)** | **用途**: 采购所有电子元器件。 <br> **检查点**: 制造商型号 (MPN) 是否唯一且准确？封装信息是否与 PCB 封装库匹配？数量、位号是否完整？是否有替代料建议？ |
| **PCBA SMT 装配** | 1. **Pick and Place File** (坐标文件/Centroid File) <br> 2. **Assembly Drawing** (装配图) <br> 3. **Assembly Notes** (装配说明) | **用途**: 指导 SMT 贴片机进行元器件贴装和后续焊接。 <br> **检查点**: 坐标原点是否明确？元器件角度是否正确？装配图是否清晰标明了极性器件（如二极管、电解电容）的方向？是否有特殊焊接要求（如手焊、特定锡膏）？ |
| **质量与测试** | 1. **DFM/DFT Report** (可制造性/可测试性报告) <br> 2. **Test Plan / Test Points List** | **用途**: 指导生产过程中的质量控制和最终功能测试。 <br> **检查点**: DFM 报告中的风险项是否已解决？测试点是否易于接触，并已在丝印层标出？ |

## 7. KPI 仪表：量化你的设计质量与效率

将文档化体系与关键绩效指标 (KPI) 结合，可以持续度量和改进您的研发流程。

*   **一次交付成功率 (First Pass Yield - FPY)**
    *   **定义**：提交给制造商的生产资料，一次性通过审核无需修改的批次数 / 总提交批次数。
    *   **目标**：> 95%。
    *   **提升方法**：严格执行内部设计评审和 DFM 检查表。

*   **DRC 零错误率 (DRC Zero-Error Rate)**
    *   **定义**：最终交付的设计文件中，经过制造商工艺规则检查后，DRC 错误数量为零。
    *   **目标**：100%。
    *   **提升方法**：在设计阶段就从制造商（如 HILPCB）获取准确的设计规则文件，并导入 EDA 工具。

*   **关键信号阻抗命中率 (Impedance Hit Rate)**
    *   **定义**：生产出的 PCB 上，通过 TDR 测试或阻抗 Coupon 测量的实际阻抗值，落在目标值 ±10% 范围内的比例。
    *   **目标**：> 90%。
    *   **提升方法**：使用专业的层叠设计和仿真工具，并在制造文件中明确要求制作阻抗 Coupon。

*   **设计迭代次数 (Design Iteration Count)**
    *   **定义**：从项目启动到最终量产，硬件版本的平均迭代次数。
    *   **目标**：持续降低。
    *   **提升方法**：加强前期需求分析和仿真，完善设计评审流程，最大化利用原型验证阶段发现问题。

## 8. DFM/DFT 清单与 HILPCB 协同服务

DFM (Design for Manufacturability) 和 DFT (Design for Testability) 是连接设计与生产的最后，也是最关键的桥梁。一个详尽的检查清单是避免“设计完美，生产抓瞎”的终极保障。

以下是一个包含超过 35 个检查项的精简版 DFM/DFT 清单，您的团队可以在每次设计发布前逐项核对。

| 类别 | 检查项 | 描述 |
| :--- | :--- | :--- |
| **PCB 制造** | 最小线宽/线距 | 是否满足 HILPCB 的标准工艺能力（如 4/4mil）？ |
| | 最小钻孔尺寸 | 机械钻孔和激光钻孔是否符合规范？ |
| | 焊盘到铜边距离 | 是否有足够的安全间距，防止短路？ |
| | BGA 焊盘设计 | Via-in-Pad 是否已按要求处理（如树脂塞孔）？ |
| | 阻焊桥 | IC 引脚之间是否有足够的阻焊桥，防止焊接连锡？ |
| | 丝印清晰度 | 丝印是否会覆盖焊盘？字符高度是否可读？ |
| | 板边间距 | 元器件距离板边是否超过 3mm，便于 V-cut 或邮票孔分割？ |
| | 金手指设计 | 是否有 45° 倒角？是否符合厚度和尺寸要求？ |
| | 拼板设计 | 拼板方式（V-cut/邮票孔）是否合理？是否添加了工艺边？ |
| | 特殊孔（槽孔/沉孔） | 是否在独立的机械层清晰标注了尺寸和公差？ |
| **PCBA 装配** | 元器件间距 | 是否有足够的空间供贴片机吸嘴和返修工具操作？ |
| | 封装库准确性 | 所有元器件的封装是否与实物 100% 匹配？ |
| | 极性标识 | 二极管、电容、IC 的一脚标识是否清晰明确？ |
| | 基准点 (Fiducial Mark) | 是否在板上或拼板边上放置了 2-3 个基准点？ |
| | 过孔与焊盘位置 | 过孔是否离焊盘太近，导致“偷锡”现象？ |
| | 大型器件支撑 | 重量大的器件（如变压器、电解电容）是否有额外的机械固定？ |
| | 底部元件高度 | 波峰焊时，底部元件高度是否有限制？ |
| | 钢网开口 | BGA、QFN 等器件的钢网开口是否需要特殊设计（如内凹）？ |
| | 红胶工艺 | 若使用红胶工艺，点胶位置是否合理？ |
| | 清洗要求 | 是否有不能水洗的器件？是否需要局部清洗？ |
